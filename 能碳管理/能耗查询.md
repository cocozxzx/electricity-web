# 平台层级结构与权限模型

## 1. 四层租户层级结构

系统采用四层租户模型，从上到下依次为：**设备厂家 → 代理商 → 项目 → 设备**。

| 层级     | 实体含义                       | 上下级关系                     | 典型数量级           |
| -------- | ------------------------------ | ------------------------------ | -------------------- |
| 设备厂家 | 平台的注册主体，生产并销售设备 | 平台顶层，管理名下所有代理商   | 1 个（或少数几个）   |
| 代理商   | 厂家授权的销售/服务商          | 属于某一厂家，管理名下多个项目 | 数十至数百个         |
| 项目     | 一个工程项目，安装了若干台设备 | 属于某一代理商，包含多台设备   | 每代理商数个至数十个 |
| 设备     | 实际采集电力数据的物理设备     | 属于某一项目，是数据最小单元   | 每项目数台至数十台   |

### 层级约束

- 设备归属唯一，每台设备属于且只属于一个项目
- 设备与项目的归属关系在设备激活/绑定时确定
- 聚合方向单向确定：设备之和 = 项目，项目之和 = 代理商，代理商之和 = 厂家全网

## 2. 权限模型

系统包含三类登录角色，各自的数据可见范围严格隔离。

| 角色   | 可见层级       | 数据范围                                           |
| ------ | -------------- | -------------------------------------------------- |
| 厂家   | 全部层级       | 所有代理商 → 所有项目 → 所有设备，无限制           |
| 代理商 | 本代理商及以下 | 仅限自己名下的项目和设备，不可见其他代理商任何数据 |
| 项目方 | 本项目及以下   | 仅限本项目及其下设备，不感知代理商和厂家层级       |

### 权限规则

- 代理商对名下项目拥有**完整可见权**，包括项目内的操作记录
- 权限隔离在**后端 API 层强制执行**，所有接口校验当前登录用户的租户归属，不能仅依赖前端菜单隐藏
- 不同角色共用同一套查询页面，根据登录身份动态渲染可用的筛选选项

## 3. 各角色查询入口与聚合逻辑

### 3.1 厂家视角

- **默认落点**：全网汇总，展示所有代理商合计用电量
- **可筛选节点**：全网 / 指定代理商 / 指定项目 / 指定设备
- **下钻路径**：全网 → 代理商 → 项目 → 设备
- **聚合逻辑**：
  - 全网总用电 = 所有代理商用电之和
  - 代理商用电 = 名下所有项目用电之和
  - 项目用电 = 名下所有设备用电之和
- **典型使用场景**："查看全网设备整体用电情况，对比各代理商能耗表现"

### 3.2 代理商视角

- **默认落点**：本代理商汇总，展示名下所有项目合计用电量

- **可筛选节点**：本代理商 / 指定项目 / 指定设备

- **下钻路径**：本代理商 → 项目 → 设备

- **聚合逻辑**

  ：

  - 代理商用电 = 名下所有项目用电之和
  - 项目用电 = 名下所有设备用电之和

- **不可见内容**：其他代理商的任何数据

- **典型使用场景**："查看我负责的各项目用电情况，识别哪个项目设备能耗异常"

### 3.3 项目方视角

**默认落点**：本项目汇总，展示名下所有项目合计用电量

**可筛选节点**：本项目 / 指定设备

**下钻路径**：本项目 → 设备

**聚合逻辑**：

- 项目用电 = 名下所有设备用电之和

**不可见内容**：代理商层级信息、其他项目的任何数据

**典型使用场景**："查看本项目总用电，以及每台设备分别的用电明细"

# 能耗查询功能设计

## 1. 功能概述

能耗查询是平台的基础数据查询模块，基于设备级电力计量数据，支持按组织层级和时间维度对用电量进行实时查看、历史追溯和多维度分析。

数据来源：设备自动上报，分钟级频率（1~15 分钟/次）。

## 2. 查询入口

三类角色共用同一套查询页面，根据登录身份动态渲染可用筛选项，不开发独立页面。权限校验在后端 API 层强制执行。

| 角色   | 默认展示                         | 可选筛选节点                            | 不可见内容           |
| ------ | -------------------------------- | --------------------------------------- | -------------------- |
| 厂家   | 全网汇总（所有代理商合计用电）   | 全网 / 指定代理商 / 指定项目 / 指定设备 | 无限制               |
| 代理商 | 本代理商汇总（名下所有项目合计） | 本代理商 / 指定项目 / 指定设备          | 其他代理商的任何数据 |
| 项目方 | 本项目汇总（本项目所有设备合计） | 本项目 / 指定设备                       | 代理商层级、其他项目 |

## 3. 时间维度

### 3.1 查询粒度与存储策略

| 查询粒度     | 适用场景               | 可查范围上限             | 存储来源                 |
| ------------ | ---------------------- | ------------------------ | ------------------------ |
| 分钟（原始） | 故障排查、异常定位     | 近 90 天，且仅设备层可用 | 设备原始上报表，滚动保留 |
| 小时         | 当日用电曲线、峰谷查看 | 近 1 年                  | 分钟数据预聚合           |
| 日           | 日报、同比环比         | 全量历史                 | 日聚合表，永久保留       |
| 月           | 月报、趋势分析、考核   | 全量历史                 | 月聚合表，永久保留       |
| 年           | 年报、长期基准对比     | 全量历史                 | 年聚合表，永久保留       |

> 分钟原始数据超 90 天后归档，不在查询界面展示，但可在碳核查支撑模块中专项调取作为原始凭证。

### 3.2 快捷时间选项

今日 / 昨日 / 本周 / 上周 / 本月 / 上月 / 本季度 / 本年 / 自定义范围

### 3.3 粒度自动推荐规则

| 查询时间范围 | 自动推荐粒度 |
| ------------ | ------------ |
| ≤ 2 天       | 小时         |
| 3 ~ 31 天    | 日           |
| 32 天以上    | 月           |

- 用户可手动覆盖自动推荐的粒度
- 分钟粒度仅在查询范围 ≤ 7 天、且选中节点为设备层时开放，区域/项目/代理商层不开放分钟粒度

## 4. 查询结果

### 4.1 核心指标

| 指标         | 单位 | 说明                                 |
| ------------ | ---- | ------------------------------------ |
| 总用电量     | kWh  | 所选时间范围内的累计用电量           |
| 平均功率     | kW   | 所选时间范围内的平均功率             |
| 最大需量     | kW   | 所选时间范围内的功率峰值及其发生时间 |
| 同比         | %    | 与去年同期相比的变化幅度             |
| 环比         | %    | 与上一统计周期相比的变化幅度         |
| 与基准值偏差 | %    | 与设定基准值相比的偏差幅度           |

### 4.2 图表展示

- 用电曲线：折线图，展示所选时间范围内的用电趋势
- 层级占比：条形图或饼图，展示下级节点的用电占比分布
- 同比环比：柱状图，对比当期与历史同期数据

### 4.3 实时看板

- 当前功率（最新一条上报值）
- 今日累计用电量
- 在线 / 离线设备数

### 4.4 设备选择交互

单项目设备数量在 100 台以上，设备筛选采用搜索框 + 分页列表的交互方式，不使用普通下拉选择器，避免一次加载过多条目影响体验。支持按设备名称或编号模糊搜索。

### 4.5 数据表格与导出

- 明细数据表，支持分页浏览
- 导出格式：Excel / CSV / PDF 报告（含图表）
- PDF 报告包含公司名称（厂家/代理商/项目方根据登录角色自动填入），不含 Logo 或盖章样式

## 5. 数据质量处理

### 5.1 处理规则

| 情形                        | 处理规则                                          | 界面表现                                 |
| --------------------------- | ------------------------------------------------- | ---------------------------------------- |
| 设备离线，数据中断          | 对应时段标记为"缺失"，不以 0 填充，不参与聚合计算 | 曲线断线显示，聚合值附加"数据不完整"标注 |
| 数据完整率 < 80%            | 聚合值仍计算但降级展示                            | 数值显示橙色 + ⚠ 图标，悬停提示完整率    |
| 瞬时功率超额定值 120%       | 标记为疑似异常点，保留原值，不剔除                | 曲线上该点高亮红色，可点击查看原始上报值 |
| 连续 5 个上报点数值完全一致 | 触发"设备冻结"告警                                | 设备状态标记"疑似冻结"，推送告警通知     |

> 缺失数据不填 0 是关键原则。填 0 会导致用电量被低估，进而影响碳排放核算、能效对标等下游功能的准确性。

### 5.2 数据完整率定义

数据完整率 = 实际上报点数 / 期望上报点数 × 100%

期望上报点数根据设备配置的上报频率计算。

# 能耗查询页面原型

## 整体布局

左侧分为两部分：功能导航栏和层级树。右侧为查询主内容区。

------

## 左侧功能导航栏

顶部显示平台名称和当前登录账号的公司名称，下方为功能菜单列表（能耗查询、能效对标、碳排放核算等），能耗查询为当前选中项高亮显示。

------

## 左侧层级树

位于功能导航栏右侧，与导航栏之间有分隔线。顶部有一个搜索框，支持按设备名称或编号模糊搜索，匹配结果在树中高亮显示。

树的结构根据角色不同：

- 厂家视角：根节点为"全网"，下级展开各代理商，点击代理商节点再展开其名下项目，点击项目节点再展开其名下设备
- 代理商视角：根节点为本代理商，下级直接展开项目和设备，看不到其他代理商
- 项目方视角：根节点为本项目，下级只展示本项目内的设备列表

默认展开规则：厂家和代理商登录后默认展开到项目层，不默认展开到设备层，避免树过长。点击任意节点立即刷新右侧内容，不需要额外点击查询按钮。当前选中节点高亮显示。

------

## 右侧页头

左侧显示当前选中节点的名称（如"全网"、"代理商 A"、"华东项目-上海"），作为当前查询范围的面包屑提示。右侧并排放置当前角色标签、导出 Excel、导出 PDF 两个按钮。

------

## 右侧筛选栏

原层级选择器去掉，只保留时间相关控件，一行横排：

- 时间范围选择器：下拉快捷选项（今日 / 本周 / 本月 / 本季度 / 本年 / 自定义），选自定义时展开日期区间输入
- 时间粒度选择器：下拉（小时 / 日 / 月 / 年），根据时间范围自动推荐，用户可手动修改
- 分钟粒度仅在选中设备层节点、且时间范围不超过 7 天时出现在粒度选项中

------

## 数据概览

三个并排数字卡片，随左侧树选中节点自动更新：

- 当前功率（kW），附环比变化标签，涨红跌绿
- 本周期累计用电量（kWh），附同比变化标签
- 设备在线数 / 总设备数，离线数量橙色标注

------

## 图表区

左右两栏：

**左栏：用电趋势折线图**

当期用实线，去年同期用虚线叠加对比。图表上方可切换"用电量"和"功率"两个指标。

**右栏：子节点占比图**

环形图展示当前选中节点下各子节点的用电占比，图例列在环形图下方，每行显示名称和百分比。

- 选中全网 → 显示各代理商占比
- 选中代理商 → 显示各项目占比
- 选中项目 → 显示各设备占比
- 选中具体设备 → 该区域改为显示该设备的当日用电柱状图

------

## 设备明细表

选中非设备层节点时展示，列出当前节点下所有设备的明细数据。选中具体设备节点时，该区域改为展示该设备的分时用电曲线（分钟或小时粒度）。

表格上方一行：左侧标题"设备明细"，右侧放搜索框（按设备名称或编号模糊搜索）和状态筛选下拉（全部 / 在线 / 离线 / 疑似冻结）。

表格列依次为：设备名称、所属项目、当前功率、今日用电量、本周期用电量、数据完整率、状态。

数据质量展示规则：

- 数据完整率低于 80% 时，该单元格橙色显示并带警告标记
- 离线设备的功率和今日用电量显示为"—"，不显示 0
- 疑似冻结的设备在状态列用橙色标注
- 有异常值的设备在对应数值单元格高亮红色，鼠标悬停时提示原因

表格底部分页，每页 20 条。
