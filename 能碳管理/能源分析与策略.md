# 能源消费分析与用能策略推荐功能设计

## 1. 功能概述

对用能单元在一个时间周期内的用电结构、用电成本、设备运行效率进行计算和分析，并基于分析结果调用 DeepSeek 大模型实时生成用能优化策略建议。

层级结构、权限模型与能耗查询保持一致。


## 2. 电价配置模块

### 2.1 配置规则

- 电价按项目维度配置，每个项目单独录入，不同项目之间互不影响
- 支持峰 / 平 / 谷三段电价结构，每段配置对应时段和单价
- 由平台管理员统一录入和维护，代理商和项目方无电价配置权限
- 电价配置支持版本管理，历史成本计算锁定当时生效的电价，不随电价更新而重算

### 2.2 配置字段

每个项目的电价配置包含以下字段：

| 字段 | 说明 |
|------|------|
| 项目名称 | 关联项目 |
| 电价类型 | 峰 / 平 / 谷 |
| 时段范围 | 每段的起止时间（如 08:00 ~ 12:00） |
| 单价（元/kWh） | 对应时段的电价 |
| 生效日期 | 该电价版本的生效起始日期 |

### 2.3 时段说明

- 一天 24 小时须被峰 / 平 / 谷三段完整覆盖，不允许时段空缺或重叠
- 支持工作日与非工作日配置不同时段划分


## 3. 用电结构分析

### 3.1 分析维度

| 维度 | 分析内容 |
|------|----------|
| 设备占比 | 各设备用电量在所属项目中的占比，识别高耗电设备 |
| 时段分布 | 24 小时用电分布热力图，识别用电集中时段 |
| 峰谷分布 | 峰 / 平 / 谷时段用电量各自占比及趋势 |
| 层级对比 | 同一代理商名下各项目之间、同一项目内各设备之间的横向用电对比 |
| 异常时段 | 非工作时间（夜间、节假日）仍有显著用电的时段标记 |

### 3.2 统计周期

支持日、周、月、季度、年及自定义时间范围。

### 3.3 展示形式

- 设备占比：条形图或饼图
- 时段分布：24 小时热力图（横轴为小时，纵轴为日期，颜色深浅代表用电强度）
- 峰谷分布：堆叠柱状图，展示各时段用电量及占比
- 层级对比：横向排名条形图，支持升降序排列
- 异常时段：在热力图上高亮标注，附异常说明


## 4. 用电成本分析

### 4.1 计算逻辑

基于设备上报的分钟级用电数据和项目配置的峰谷平电价，按时段匹配计算各时段用电成本，汇总得到周期内总用电成本。

```
各时段用电成本 = 该时段用电量（kWh）× 对应时段单价（元/kWh）
周期总成本 = 峰段成本 + 平段成本 + 谷段成本
```

### 4.2 分析内容

| 分析项 | 说明 |
|--------|------|
| 周期总用电成本 | 统计周期内的总电费支出 |
| 峰 / 平 / 谷成本占比 | 各时段电费在总成本中的比例 |
| 成本趋势 | 按日 / 月展示成本变化曲线，附同比环比 |
| 高成本设备排行 | 按设备维度计算各设备的用电成本，排名展示 |
| 峰段用电占比 | 高电价时段用电量占总用电量的比例，作为优化参考 |

### 4.3 展示形式

- 周期总成本：数字卡片，附同比环比变化标签
- 峰 / 平 / 谷成本占比：堆叠柱状图或饼图
- 成本趋势：折线图
- 高成本设备排行：条形图，支持点击下钻查看设备用电曲线

### 4.4 数据依赖说明

未配置电价的项目，成本分析模块显示"暂未配置电价，请联系平台管理员"，不显示估算值。


## 5. 设备运行效率分析

### 5.1 数据来源

- 实际功率：设备实时上报
- 额定功率：设备上报的铭牌参数字段

### 5.2 核心指标

| 指标 | 计算方式 | 说明 |
|------|----------|------|
| 负载率 | 实际功率 ÷ 额定功率 × 100% | 反映设备当前运行负荷水平 |
| 平均负载率 | 统计周期内负载率的均值 | 反映设备整体运行效率 |
| 轻载时长占比 | 负载率 < 30% 的时长 ÷ 总运行时长 | 识别长期低效运行设备 |
| 超载时长占比 | 负载率 > 100% 的时长 ÷ 总运行时长 | 识别超负荷运行设备 |
| 空载时长占比 | 功率 < 5% 额定功率的时长 ÷ 总运行时长 | 识别怠速运行设备 |

### 5.3 效率评级

| 评级 | 负载率区间 | 含义 |
|------|------------|------|
| 优 | 70% ~ 100% | 运行状态良好 |
| 良 | 50% ~ 70% | 运行效率一般 |
| 差 | 30% ~ 50% | 长期低效，建议排查 |
| 异常 | < 30% 或 > 100% | 轻载 / 超载，需关注 |

### 5.4 展示形式

- 设备效率排行榜：按平均负载率升序排列，快速定位低效设备
- 单设备负载率曲线：选中设备后展示统计周期内的负载率折线图
- 效率分布图：展示当前层级下各评级设备数量占比


## 6. AI 用能策略推荐

### 6.1 技术方案

使用 **DeepSeek** 大模型生成用能优化策略建议。用户在分析页面点击"生成建议"按钮后，系统实时调用 DeepSeek API，将当前分析结果作为上下文输入，返回结构化策略建议文本，流式展示。

### 6.2 输入上下文

调用 DeepSeek 时，系统自动将以下分析结果作为 Prompt 上下文传入：

- 当前查询层级和统计周期
- 用电结构摘要（高耗电设备 Top 5、峰谷分布比例、非工作时间用电占比）
- 用电成本摘要（总成本、峰段成本占比、高成本设备 Top 3）
- 设备效率摘要（低效设备数量及占比、超载设备列表、平均负载率）
- 同比 / 环比变化趋势

### 6.3 输出内容

DeepSeek 返回的建议按以下类型组织展示：

| 建议类型 | 内容说明 |
|----------|----------|
| 用电调度优化 | 基于峰谷分布，提出错峰用电、调整设备启停时间等建议 |
| 成本压降建议 | 针对峰段用电占比高的情况，建议将可调负荷迁移至谷段 |
| 低效设备处理 | 针对轻载 / 空载设备，建议合并运行、停机待机或检修排查 |
| 超载设备预警 | 针对超载设备，提示安全风险并建议负荷调配 |
| 非工作时间管控 | 针对非工作时间异常用电，建议排查空载运行或优化自动断电策略 |
| 清洁能源建议 | 根据当前用电规模，提示绿电 / 绿证采购参考用量 |

### 6.4 交互设计

- 用户点击"生成建议"按钮后，页面显示加载状态，DeepSeek 返回后流式展示建议内容
- 建议内容只读展示，不收集用户反馈
- 每次点击均重新调用 DeepSeek，基于当前最新分析数据生成，不缓存历史建议
- 建议内容支持随分析报告一并导出为 PDF

### 6.5 免责说明

页面和报告中须标注："以上建议由 AI 大模型根据用电数据自动生成，实际执行前请结合现场工况进行专业判断。"


## 7. 功能边界说明

| 项目 | 是否支持 | 说明 |
|------|----------|------|
| 用电结构分析 | 支持 | 基于设备上报电力数据完整实现 |
| 用电成本分析 | 支持 | 依赖平台管理员配置各项目峰谷平电价 |
| 设备运行效率分析 | 支持 | 依赖设备上报的额定功率字段 |
| 综合能效分析 | 不支持 | 缺少产量 / 产值数据，无法计算单位产品能耗 |
| AI 策略推荐 | 支持 | 使用 DeepSeek 大模型实时生成 |
| 策略执行反馈 | 不支持 | 本期只展示建议，不收集执行结果 |
